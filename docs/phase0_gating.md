# Phase 0 Gate: Parity Baseline Readiness

Overview
- Phase 0 is the stabilization and scaffolding phase to enable Phase 1 parity work.
- This document defines the gates, acceptance criteria, and sign-off workflow for Phase 0.

Gate Criteria (Pass/Fail)
- Gate 0.1: Baseline parity artifacts created: Phase 0 Runbook, parity scorecard skeleton, 8 Phase 0 tickets, crosswalk skeletons.
- Gate 0.2: Phase 0 Runbook executed in a reproducible way; artifacts generated (phase0-baseline artifacts + run outputs).
- Gate 0.3: Crosswalks populated for at least two axes (OpenClaw, AgentZero) with filled rows, Hermes crosswalk and OpenFang opened as templates.
- Gate 0.4: Phase 0 tickets moved to In Progress/Done in tracker, with owners assigned.
- Gate 0.5: Parity Scorecard populated and baseline gating documented.

Sign-off
- Phase 0 Owner (TBD) signs off and moves to Phase 1.

Mitigations and Escalation
- If Gate 0.5 cannot be satisfied within the agreed window, escalate to the governance board with a concrete corrective plan.
