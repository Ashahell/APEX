# Migration Guide: OpenFang → APEX

Intent
- Describe how to migrate components from OpenFang into the APEX codebase, focusing on Hands, MCP, and memory components.

Scope
- MVP mapping for Computer Use Hand, MCP tool registry, and memory scaffolds.
- Identify gaps and plan for the first wave of ported components.

- Migration steps (high level)
- Step 0: Readiness assessment and quick-start mapping
- Step 1: Map Hands/ Hand runner to APEX code structure
- Step 2: Map MCP tool registry to APEX's internal registry
- Step 3: Port the memory scaffolds (bounded memory, index) into APEX MVP
