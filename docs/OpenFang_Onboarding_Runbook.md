# OpenFang Onboarding Runbook (ACME MVP)

- Objective: Provide a practical onboarding runbook for OpenFang adoption in APEX. It should be used by teams to onboard to the Hands MVP and OpenFang primitives.
- Scope: onboarding of Hands (Computer Use) MVP, MCP scaffolding, memory scaffolds, and minimal API surface.
- Prerequisites: governance cadence established; baseline repo changes merged; MVP API surface running locally.
- Steps:
- 1. Confirm MVP scope and ownership (Hands, MCP, Memory)
- 2. Spin up local MVP server (apex_router_bin) and validate endpoints
- 3. Create a Hands MVP task and verify lifecycle (start/status/stream/logs)
- 4. Validate memory/embedding wiring (add a test embedding, verify query path)
- 5. Validate the security baseline (HMAC, timestamp window placeholder, audit log placeholder)
- 6. Collect feedback and update crosswalk and governance docs
- 7. Prepare pilot for 2–3 teams
