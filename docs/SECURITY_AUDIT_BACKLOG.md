Security Audit Backlog (Phase 1)
===============================

- Critical: Build secrets exposure, insecure defaults, or privilege escalation in container runtime.
- High: Dependency vulnerabilities, misconfigured TLS, weak authentication, insufficient logging/audit accuracy.
- Medium: Obsolete dependencies, lint/security warnings in code, insecure CI gates.
- Low: Non-critical hardening gaps, documentation gaps.

Backlog Items (to be filled as auditors report findings)
- [ ] Item description, severity, asset/component, owner, status
- [ ] Phase 1 Outreach: Send initial RFP emails to shortlist vendors; track responses and follow-ups

- Phase 1 backlog actions (to be scheduled):
  - [ ] Issue vendor outreach: prepare and send RFP to 2-3 qualified security firms aligned with SOW.
  - [ ] Onboard vendor: NDA/signature, access provisioning, security clearances.
  - [ ] Schedule kickoff meeting and define audit milestones.
  - [ ] Map audit findings to remediation backlog (tie to SECURITY_AUDIT_SOW.md).
  - [ ] Prepare remediation review and assign owners.
